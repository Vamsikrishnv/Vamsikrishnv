## Who I am

I build like a storyteller: I start with the human problem, follow the constraints, and ship the simplest system that reliably moves the outcome.

Over time my mindset shifted from “build something impressive” to “build something that survives reality”:
- Reduce complexity before adding intelligence  
- Prefer repeatable workflows over clever one-offs  
- Make failures visible, debugging easy, and iteration cheap  
- Optimize for trust: reliability, evaluation, and observability  

Clarity beats cleverness.
Systems beat hacks.

AI systems
- Retrieval-Augmented Generation (RAG) architectures
- Document processing pipelines (parsing, chunking, validation, confidence)

Full-stack engineering
- Backend-first architecture and API design
- Async workflows and orchestration
